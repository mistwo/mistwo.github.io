2021.5.9 毕业答辩

发表于 2021-05-09 更新于 2021-06-03 分类于 摸鱼日志

前两天老师在群里发了一辩名单，我的名字在上面，第四个，之后我开始莫名紧张，没有食欲，口水鸡都比平时少吃半只，现在想起来有点可惜。

怎么硕呢，这次答辩还蛮有戏剧感的，有几个场景不说永生难忘，起码半年难忘吧。

ACTION 01

路人甲：尊敬的各位老师同学大家好……我的研究背景是……

y老师（低头翻论文）：少废话，讲重点

ACTION 02

路人乙：哔哔哔

y老师（举起手机）：你论文抄的这一篇？一模一样，别讲了，下去！

ACTION 03

路人丙：……感谢各位老师聆听！

y老师：我就直说，你们用 Spring Boot 的全是买的，诶，刚好这个是我的强项——你把 xxx 的配置文件写在黑板上。

路人丙：我是自己做的，xxx 自动生成……

y老师：你总记得一点吧？把你知道的写在黑板上

路人丙转身写字

y老师：全错！

ACTION 04

路人丁：……感谢老师聆听！

w老师（看着论文直摇头）：格式有很大问题

y老师：格式这么差，是我审核过的吗？

路人丁：是你审核过的……

y老师：你的意思是怪我喽？

路人丁：可能我交错了一份，打印了上个版本的格式

y老师：你又说是给我审核过的最后一个版本，又说是上一个版本，什么意思？

路人丁：emmm……

ACTION 05

路人戊：……感谢老师聆听！

y老师：你的工作是什么，你写了算法吗？

路人戊：没，调用网上的算法啊……

y老师：算法是别人的，那你建立了模型？

路人戊：没，用的前人的模型……

y老师：那你不是啥也没干？

路人戊：我训练了数据集

y老师：训练是机器做的，意思是你坐着等结果也算工作？

路人戊：emmm……

ACTION 06

我提前到达教室，把 PPT 复制到电脑，却无法打开。意识到教室里的古董电脑性能堪忧，挺不住我 50MB 的PPT，我赶紧回去修改，终于在答辩开始前把大小压缩了一半。

答辩组长（走神中）：……下一个

我从容上台，双击答辩 PPT，举手投足之间……电脑卡死了……

我（尴尬 max）：可能是 PPT 太大了

答辩组长（不关我事啊）：你重新打开看看

y老师：PPT 搞这么大干嘛

我：放了几张系统演示截图，也没多大（电脑太老了）

y老师：计算机专业的学生这都搞不定，重启试试

我：重启的话，文件就全没了

y老师：emmm……你先调到最后一个答辩

A FEW MOMENT LATER

我：bbb……感谢老师聆听！

y老师：你用到了 Spring 框架？介绍一下这个框架

我：用到了 Spring 里的 Spring JDBC

y老师：介绍一下

我：因为原始的 JDBC 操作比较繁琐，Spring JDBC 封装了一些重复的操作，减少了 web 开发中的工作量。

z老师：说一下保存图片的信息

我：图片表包含图片 id、所属类型、图片路径、详细描述、添加时间等字段

z老师：也就是说图片还是保存在本地？

我：文件已经上传到服务器了，图片路径是服务器里图片的位置

w老师：你这个图片的格式还有问题，题注和图片没在同一页，回去调整一下

我：好的好的

经验总结：

答辩之前要熟悉论文，厘清全文逻辑，同时把握工作的重难点

答辩的时候要冷静，态度要积极，把自己的想法有条理地表达出来。过度的紧张情绪会让我们遗漏重要的细节。
